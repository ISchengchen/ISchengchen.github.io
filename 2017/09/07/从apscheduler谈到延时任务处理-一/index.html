从apscheduler谈到延时任务处理

发表于 2017-09-07

Advanced Python Scheduler (APScheduler) is a Python library that lets you schedule your Python code to be executed later, either just once or periodically. You can add new jobs or remove old ones on the fly as you please. If you store your jobs in a database, they will also survive scheduler restarts and maintain their state. When the scheduler is restarted, it will then run all the jobs it should have run while it was offline。

这是官方文档的介绍，目前apscheduler的最新版本为3.3.1，从2到3增加很多异步框架的支持，比如gevent,tornado,twisted,asyncio等。为了把脉络看的更清晰，我们看2.1版本的代码。

这是项目结构。我会把对源码的解读以注释的形式表现。

jobstore:负责将任务进行持久化，默认是存在内存里。

trigger:包含了三种触发器，cron方式，固定时间间隔方式和指定日期方式，主要是计算下次任务触发的时间。

scheduler:主模块，负责任务的调度

job:定义了任务

events：定义了一些事件

threadpool:一个简易的线程池
